03 สายพานประมวลผล

นี่คือหัวใจจริงๆ ของระบบ — ทุกขั้นอยู่ใน capcut_core.py (1003 บรรทัด, 40 ฟังก์ชัน) ลำดับนี้เป็นลำดับจริง ข้ามขั้นไม่ได้ หน่วยเวลาใน draft CapCut = ไมโครวินาที

ถอดเสียง

04 สองเส้นทางออก

ต่างกันที่ปลายทาง — และนี่คือหัวใจของการแจก user

ขั้น 01–06 ใช้ร่วมกันทั้งคู่ ต่างกันแค่ขั้น 07 ปลายทาง ตัวนี้กำหนดว่า feature ไหนขึ้น cloud ได้ ไหนต้องอยู่ในเครื่อง user

เส้นทาง A build_capcut.py

→ %LOCALAPPDATA%\CapCut\…\com.lveditor.draft\<ชื่องาน>
  • เขียนไฟล์ draft เข้าโฟลเดอร์ CapCut ตรงๆ
  • เปิด CapCut เห็นโปรเจกต์เลย ตัดต่อต่อได้
  • ได้ซับสวย + pop animation + zoom transition
  • ขึ้น cloud ไม่ได้ — ต้องรันในเครื่องที่มี CapCut

เส้นทาง B process_easycut.py

→ CAPCUT_Easy_CUT.zip (mp4 + srt/ass + transcript)
  • ตัด dead air + ทำซับ แล้วแพ็กเป็น .zip
  • ไม่พึ่ง CapCut ในเครื่อง เอาไปเปิดที่ไหนก็ได้
  • เหมาะกับทำเป็นเว็บ SaaS (ต้องมี GPU/คิวงาน)
  • ไม่ได้โปรเจกต์ CapCut พร้อมตัด (ได้แค่ไฟล์)

07 ชั้น Portability · งานล่าสุด

สิ่งที่แก้เพื่อให้รันบนเครื่อง user คนอื่นได้

เดิมเอนจินผูกกับเครื่อง dev (path ส่วนตัว · ต้องมี draft เดิม · โมเดลหนัก · พึ่ง PATH) 4 จุดนี้แก้ให้ระบบ "ย้ายเครื่องได้" — ผ่านการทดสอบจำลองเครื่องเปล่าด้วยคลิปพูดไทยจริง 38 วินาทีแล้ว

STEP 1 · Template

Bundled template + token

เดิม: ต้องมี draft เดิมในเครื่อง ไม่งั้น error
ฝัง template_draft/ ที่ล้าง path ส่วนตัวออก แทนด้วย __LOCALAPPDATA__ แล้ว _substitute_tokens() แปลงเป็น path จริงตอนสร้าง
STEP 2 · Resource

pop/effect path แบบ dynamic

เดิม: hardcode C:/Users/onpre/…
resolve_effect_cache(id) หา cache ตามเครื่อง user เอง — ไม่มีคืน "" ให้ CapCut โหลดเองด้วย resource_id + ปิดได้ด้วย pop_anim_on
STEP 3 · Whisper

โมเดลเบาลง + auto GPU

เดิม: large-v3 บน CPU (~3GB, ช้ามาก)
default medium · device=auto (เจอ GPU ใช้เลย ไม่มี fallback CPU) · ปรับผ่าน env · มี progress %
STEP 4 · Tools

ffmpeg/font resolve + เช็กติดตั้ง

เดิม: เรียก ffmpeg ลอยๆ พึ่ง PATH · font hardcode
FFMPEG/FFPROBE ผ่าน env (เตรียม bundle) · resolve_font() fallback · ensure_ffmpeg/ensure_capcut() เช็กก่อนทำงาน
