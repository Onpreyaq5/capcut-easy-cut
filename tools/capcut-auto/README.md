# 🎬 capcut-auto — เอนจินของ CAPCUT Easy CUT

เอาคลิปดิบหรือคลิปทั้งชุดมา → ได้ไฟล์/โปรเจกต์ **CapCut** พร้อมทำต่อทันที
โดยระบบทำให้อัตโนมัติ:

- ✂️ **ตัด dead air** (ช่วงเงียบ) ทุกคลิป ให้กระชับ
- 🔗 **รวมทุกคลิป** เป็นไทม์ไลน์เดียว (เรียงตามชื่อไฟล์ 01, 02, 03…)
- 📝 **Smart Karaoke Subtitle** จาก Whisper word timestamps แบ่งเป็นคำ/วลีสั้นที่อ่านทัน ไม่แตกพยางค์เร็วเกินไป
- 🎨 **เน้น Keyword** ให้ตัวใหญ่ หนา สีเด่น และส่งคำรองไปซ้าย/ขวาใน `subtitles_styled.ass` หรือ draft CapCut
- ✨ อนิเมชัน **pop** + 🔀 **ทรานสิชันซูม** ตรงรอยต่อฉาก
- เขียนไฟล์โปรเจกต์ลงโฟลเดอร์ CapCut ให้เลย **ไม่ต้อง import เอง**

---

## ครั้งแรก (ติดตั้ง)
ดับเบิลคลิก **`⚙️ ติดตั้งครั้งแรก.bat`** (ติดตั้ง faster-whisper + pythainlp + python-crfsuite + requests)
ต้องมีในเครื่องก่อน: **Python 3.10+**, **ffmpeg** (`winget install Gyan.FFmpeg`), **CapCut** (เปิดสัก 1 ครั้งก่อนใช้งาน)

ไม่ต้องมีโปรเจกต์ CapCut เดิมมาก่อน — เอนจินมี template สำเร็จรูป (`template_draft/`) แจกมาให้ในตัวแล้ว

## ใช้งาน — 2 วิธี (เอนจินเดียวกัน)

### วิธี A — หน้าเว็บ CAPCUT Easy CUT
1. เปิดเว็บด้วย `▶️ เปิดเว็บ CAPCUT Easy CUT.bat`
2. ลากคลิปลงหน้าแรก (หน้า "ตัดออโต้")
3. (ไม่บังคับ) พิมพ์ข้อความ Hook เขียวช่วงเปิดคลิป, เปิด "AI ตรวจแก้ภาษาไทย" แล้วใส่ key ฟรี (Groq/Cerebras/OpenRouter/Gemini)
4. กด **ดาวน์โหลดแพ็กเกจ** เพื่อได้วิดีโอหลังตัด + `subtitles.srt` + `subtitles_styled.ass`
5. หรือกด **สร้างใน CapCut** เพื่อเขียน draft เข้า CapCut ในเครื่องโดยตรง

### วิธี B — ดับเบิลคลิกเอนจิน
1. เอาคลิปทั้งชุดใส่โฟลเดอร์เดียว ตั้งชื่อ `01.mp4`, `02.mp4`, … ตามลำดับเล่า
2. (ถ้ามีบท) วางไฟล์ `script.json` ในโฟลเดอร์นั้น — ดู `example.script.json`
3. ลากโฟลเดอร์นั้นไปวางบน **`🎬 สร้างซับ CapCut.bat`** (หรือดับเบิลคลิกแล้วพิมพ์ที่อยู่)
4. รอสักครู่ → ปิด CapCut แล้วเปิดใหม่ → เลือกโปรเจกต์ตัวบนสุด

> ปุ่มในเว็บทำงานผ่าน API route `src/app/api/capcut/build` ที่เรียกเอนจินนี้ — ใช้ได้เฉพาะรันในเครื่อง (บน Vercel เขียนไฟล์ CapCut ไม่ได้)

## คำสั่งตรง (ขั้นสูง)
```bash
python build_capcut.py --clips "โฟลเดอร์คลิป" --name "ชื่องาน" [--script script.json] [--no-transitions]
  [--hook "ข้อความ"] [--llm-provider groq --llm-key XXX --llm-model llama-3.3-70b-versatile]
```
ปรับ whisper ได้ผ่าน env: `EASYCUT_WHISPER_MODEL` (small/medium/large-v3, ดีฟอลต์ medium), `EASYCUT_WHISPER_DEVICE` (auto/cuda/cpu)

## ปรับแต่ง — `brand.json`
- `green_kw` : คำที่ให้เป็นสีเขียว (เช่น CAPCUT, Easy CUT, ซับ)
- `yellow_kw`: คำที่ให้เป็นสีเหลือง (ผล/หน่วย เช่น ซีซี ลิตร กระสอบ ไร่)
- `corrections`: แทนคำที่ whisper ถอดเพี้ยน เช่น `{"แคปคัด":"แคปคัท"}`
- `font_size` (ดีฟอลต์ 12), `line_chars` (ความกว้างบรรทัด 15), `y_pos` (ตำแหน่งซับ)
- `pop_anim_on`: ปิดได้ (`false`) ถ้า pop animation มีปัญหาบางเครื่อง

## ไฟล์ในโฟลเดอร์นี้
| ไฟล์ | หน้าที่ |
|---|---|
| `capcut_core.py` | เอนจิน (dead air, ซับ, สี, pop, ซูม, AI ตรวจภาษาไทย, เขียน draft) |
| `process_easycut.py` | สร้างแพ็กเกจ ZIP-ready: วิดีโอหลังตัด + smart subtitles + transcript |
| `build_capcut.py` | ตัวรวมงาน: คลิปหลายไฟล์ → โปรเจกต์เดียว |
| `template_draft/` | เทมเพลตโปรเจกต์ CapCut สำเร็จรูป แจกไปกับแอป (ไม่ต้องมีโปรเจกต์เดิม) |
| `brand.json` | ตั้งค่าแบรนด์/สี/คำแก้ |
| `🎬 สร้างซับ CapCut.bat` / `⚙️ ติดตั้งครั้งแรก.bat` | ตัวกดใช้งาน |

## ⚠️ สำคัญ: ห้ามมีไฟล์งานอยู่ในโฟลเดอร์ที่ซิงก์คลาวด์ (OneDrive/Dropbox/Google Drive)
ถ้าไฟล์วิดีโอที่ CapCut อ้างอิงอยู่ในโฟลเดอร์ที่ซิงก์คลาวด์ โปรแกรมซิงก์อาจแปลงไฟล์เป็น "cloud-only placeholder"
ทีหลัง (เพื่อประหยัดที่ดิสก์) ทำให้ CapCut เปิดไม่ได้ — ขึ้นเป็นคลิปสีแดง/หาไฟล์ไม่เจอ

**เอนจินนี้ป้องกันให้อัตโนมัติแล้ว** (ตั้งแต่เวอร์ชันนี้เป็นต้นไป): ไฟล์รวมที่เรนเดอร์ (`combined.mp4`) จะถูกย้าย
ไปเก็บที่ `%LOCALAPPDATA%\CAPCUT_Easy_CUT\_capcut_work\` เสมอโดยอัตโนมัติ (ผ่าน `safe_work_dir()`) ไม่ว่าโฟลเดอร์คลิป
ต้นทางที่ผู้ใช้ลากมาจะอยู่ใต้ OneDrive/Desktop หรือที่ไหนก็ตาม — ผู้ใช้ไม่ต้องทำอะไรเพิ่ม
